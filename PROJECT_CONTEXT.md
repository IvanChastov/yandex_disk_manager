# ПРОЕКТ: Менеджер Яндекс.Диска (Yandex.Disk Manager)

## 📋 ОПИСАНИЕ ПРОЕКТА

Десктопное приложение с графическим интерфейсом для управления файлами на Яндекс.Диске в рамках небольшой строительной компании. Приложение позволяет отслеживать изменения файлов (в том числе сделанные напрямую через диск), назначать теги, работать с ролями пользователей.

### Цель разработки
Выпускная квалификационная работа (ВКР) для университета + создание реального рабочего инструмента.

### Ключевые особенности
- Отслеживание изменений, сделанных напрямую через Яндекс.Диск (вне приложения)
- Система тегов с сохранением при перемещении/переименовании файлов
- Ролевая модель (Admin, Manager, Viewer)
- Лента последних изменений с индикацией источника (через приложение / напрямую)

## 🎯 СТРАТЕГИЯ РАЗВИТИЯ ПРОЕКТА (на основе анализа ограничений API)

### Выявленное ограничение
API Яндекс.Диска **не предоставляет доступ к расшаренным папкам** через токен пользователя, который не является владельцем. Это критично для сценария строительной компании, где файлы хранятся в общей папке.

## 🎯 ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

### Роли пользователей
| Роль | Права |
|------|-------|
| Admin | Полный доступ: просмотр, загрузка, скачивание, удаление, управление тегами, управление пользователями |
| Manager | Просмотр, загрузка, скачивание, назначение тегов |
| Viewer | Только просмотр и скачивание |

### Основные возможности
1. **Авторизация в Яндекс.Диске** через OAuth (получение и сохранение токена)
2. **Просмотр файлов и папок** с диска (как в проводнике)
3. **Работа с файлами**:
   - Скачивание на локальный компьютер
   - Загрузка с локального компьютера
   - Удаление (для Admin)
   - Создание папок
4. **Система тегов**:
   - Создание цветных тегов
   - Назначение тегов файлам (множественный выбор)
   - Фильтрация файлов по тегам
   - Сохранение тегов при перемещении/переименовании файлов
5. **Мониторинг изменений**:
   - Автоматическая проверка каждые 5-15 минут
   - Обнаружение изменений, сделанных напрямую через диск
   - Лента последних изменений с указанием источника
6. **Уведомления**:
   - Цветовая индикация в ленте (зелёный - через приложение, оранжевый - напрямую)
   - Всплывающие уведомления о важных изменениях (опционально)

# core/models.py (актуальная версия)

class User(AbstractUser):
    """Расширенная модель пользователя"""
    ROLE_CHOICES = [
        ('admin', 'Administrator'),
        ('manager', 'Manager'),
        ('viewer', 'Viewer'),
    ]
    role = models.CharField(max_length=20, choices=ROLE_CHOICES, default='viewer')
    yandex_token = models.TextField(blank=True, null=True)  # Токен доступа
    last_sync = models.DateTimeField(null=True, blank=True)

class Tag(models.Model):
    """Тег для маркировки файлов"""
    name = models.CharField(max_length=100, unique=True)
    color = models.CharField(max_length=7, default='#3498db')
    created_by = models.ForeignKey(User, on_delete=models.SET_NULL, null=True)
    created_at = models.DateTimeField(auto_now_add=True)

class File(models.Model):
    """Информация о файле на Яндекс.Диске"""
    yandex_id = models.CharField(max_length=255, unique=True)
    name = models.CharField(max_length=255)
    path = models.TextField()
    type = models.CharField(max_length=50)  # file/dir
    mime_type = models.CharField(max_length=100, blank=True, null=True)
    size = models.BigIntegerField(default=0, null=True)
    created_at = models.DateTimeField()
    modified_at = models.DateTimeField()
    tags = models.ManyToManyField(Tag, related_name='files', blank=True)
    created_by = models.ForeignKey(User, on_delete=models.SET_NULL, null=True, blank=True, related_name='created_files')
    modified_by = models.ForeignKey(User, on_delete=models.SET_NULL, null=True, blank=True, related_name='modified_files')
    last_synced = models.DateTimeField(auto_now=True)

class ChangeLog(models.Model):
    """Лог изменений файлов"""
    CHANGE_TYPES = [
        ('created', 'Создан'),
        ('modified', 'Изменен'),
        ('moved', 'Перемещен'),
        ('deleted', 'Удален'),
    ]
    SOURCE_TYPES = [
        ('app', 'Через приложение'),
        ('direct', 'Напрямую через диск'),
        ('unknown', 'Неизвестно'),
    ]
    
    file = models.ForeignKey(File, on_delete=models.CASCADE, null=True, blank=True)
    file_path = models.TextField()
    change_type = models.CharField(max_length=20, choices=CHANGE_TYPES)
    source = models.CharField(max_length=20, choices=SOURCE_TYPES, default='app')
    changed_by = models.ForeignKey(User, on_delete=models.SET_NULL, null=True, blank=True)
    changed_at = models.DateTimeField()
    detected_at = models.DateTimeField(auto_now_add=True)
    old_path = models.TextField(blank=True)
    
    class Meta:
        ordering = ['-changed_at']
