# ПРОЕКТ: Менеджер Яндекс.Диска (Yandex.Disk Manager)

## 📋 ОПИСАНИЕ ПРОЕКТА

Десктопное приложение с графическим интерфейсом для управления файлами на Яндекс.Диске в рамках небольшой строительной компании. Приложение позволяет отслеживать изменения файлов (в том числе сделанные напрямую через диск), назначать теги, работать с ролями пользователей.

### Цель разработки
Выпускная квалификационная работа (ВКР) для университета + создание реального рабочего инструмента.

### Ключевые особенности
- Отслеживание изменений, сделанных напрямую через Яндекс.Диск (вне приложения)
- Система тегов с сохранением при перемещении/переименовании файлов
- Ролевая модель (Admin, Manager, Viewer)
- Лента последних изменений с индикацией источника (через приложение / напрямую)
- **Корпоративный режим:** единый токен для всей компании, вход по логину/паролю

## 🎯 СТРАТЕГИЯ РАЗВИТИЯ ПРОЕКТА (на основе анализа ограничений API)

### Выявленное ограничение
API Яндекс.Диска **не предоставляет доступ к расшаренным папкам** через токен пользователя, который не является владельцем. Это критично для сценария строительной компании, где файлы хранятся в общей папке.

## 🎯 ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

### Роли пользователей
| Роль | Права |
|------|-------|
| Admin | Полный доступ: просмотр, загрузка, скачивание, удаление, управление тегами, управление пользователями, смена токена |
| Manager | Просмотр, загрузка, скачивание, назначение тегов |
| Viewer | Только просмотр и скачивание |

### Основные возможности
1. **Вход в приложение** — по логину/паролю (Django User)
2. **Просмотр файлов и папок** с диска (как в проводнике)
3. **Работа с файлами**:
   - Скачивание на локальный компьютер
   - Загрузка с локального компьютера
   - Удаление (для Admin)
   - Создание папок
4. **Система тегов**:
   - Создание цветных тегов
   - Назначение тегов файлам (множественный выбор)
   - Фильтрация файлов по тегам
   - Сохранение тегов при перемещении/переименовании файлов
5. **Мониторинг изменений**:
   - Автоматическая проверка каждые 5 минут
   - Обнаружение изменений, сделанных напрямую через диск
   - Лента последних изменений с указанием источника
6. **Поиск файлов**: по имени и по тегам
7. **Фильтр по дате**: в ленте изменений (всё, сегодня, неделя, месяц)
8. **Экспорт отчёта**: выгрузка истории изменений в CSV
9. **Предпросмотр файлов**:
   - Изображения (jpg, png, gif, bmp, webp)
   - Текстовые файлы (txt, py, json, xml, html, css, js, md, log)
   - PDF
   - CAD (dwg, dxf)
10. **Администрирование**:
    - Управление пользователями (создание, редактирование, удаление)
    - Смена корпоративного токена

🐛 ИЗВЕСТНЫЕ ПРОБЛЕМЫ И ОГРАНИЧЕНИЯ
Доступ к расшаренным папкам — API Яндекс.Диска не позволяет получить доступ к расшаренным папкам через чужой токен.

Глубокий обход файловой структуры — мониторинг рекурсивно обходит все папки, что для больших дисков может быть ресурсоёмким.

Локальная БД — при использовании на нескольких ПК теги и история не синхронизируются. Для масштабирования требуется централизованная БД (PostgreSQL).
