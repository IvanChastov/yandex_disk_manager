# Руководство пользователя

## Первый запуск

1. Запустите `YandexDiskManager.exe`
2. В меню выберите **Файл → Авторизация**
3. Нажмите **"Получить код"** — откроется страница Яндекса
4. Войдите в свой аккаунт и нажмите **"Разрешить"**
5. Скопируйте код из адресной строки (после `code=`)
6. Вставьте код в поле и нажмите **"Получить токен"**

## Основные действия

| Действие | Как сделать |
|----------|-------------|
| Просмотр файлов | Двойной клик по папке |
| Назад | Кнопка "Назад" вверху |
| Скачать файл | Правый клик → "Скачать" |
| Загрузить файл | Меню "Диск" → "Загрузить файл" |
| Создать папку | Меню "Диск" → "Создать папку" |
| Удалить файл | Правый клик → "Удалить" (только Admin) |

## Горячие клавиши

- `F5` — обновить
- `Ctrl+F` — поиск
- `Ctrl+D` — скачать выбранный файл
- `Delete` — удалить
- `Ctrl+Q` — выход

## Роли

- **Admin** — всё доступно
- **Manager** — просмотр, загрузка, теги (без удаления)
- **Viewer** — только просмотр



## Работа с общей папкой компании

### Ограничение API Яндекс.Диска
API Яндекса **не даёт доступ к расшаренным папкам** через токен пользователя, который не является владельцем папки. Если вам нужен доступ к общей папке компании, используйте один из вариантов ниже.

### Вариант 1: Использовать токен владельца (рекомендуется)
1. Попросите владельца общей папки (администратора) выполнить авторизацию через приложение:
   - **Файл → Авторизация**
   - Получить код, разрешить доступ
   - Скопировать полученный токен
2. Сохраните этот токен в файле `settings.json` (в папке `%APPDATA%/YandexDiskManager/`) или передайте владельцу для использования

### Вариант 2: Создать служебный аккаунт
1. Администратор создаёт отдельный аккаунт (например, `disk@l-engeneering.ru`)
2. Выдаёт этому аккаунту доступ к общей папке
3. Получает токен от этого аккаунта через приложение
4. Использует этот токен для всех операций с общей папкой

### Как понять, какой токен используется
При запуске приложения в строке состояния отображается имя текущего пользователя. Если это имя владельца папки — доступ будет работать. Если нет — папка будет недоступна.
