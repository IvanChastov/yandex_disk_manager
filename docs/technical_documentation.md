## Ограничения API

### Доступ к расшаренным папкам
API Яндекс.Диска не позволяет получить доступ к расшаренным папкам через токен пользователя, который не является владельцем папки. Это подтверждено экспериментально (скрипт `check_shared_api.py`) и документацией Яндекса.

### Решения
| Вариант | Описание | Плюсы | Минусы |
|---------|----------|-------|--------|
| **Токен владельца** | Использовать токен пользователя, который создал общую папку | Простота, не требует изменений в коде | Зависит от конкретного сотрудника |
| **Служебный аккаунт** | Создать отдельный аккаунт и дать ему доступ | Независимость от сотрудников, можно контролировать права | Требует участия администратора |
| **Собственное S3-хранилище** | Развернуть MinIO и перенести данные | Полный контроль, нет зависимости от Яндекса | Высокие затраты на внедрение |

### Код для проверки доступа
```python
# Проверка, какой токен используется
from core.yandex.storage import get_current_user, get_token_for_user
username = get_current_user()
token = get_token_for_user(username)
print(f"Текущий пользователь: {username}")
print(f"Токен: {token[:20]}...")


## Масштабирование и централизация

### Текущая архитектура (для малых компаний)
- SQLite — файловая БД на компьютере пользователя
- Подходит для 1-5 сотрудников
- Простота развёртывания

### Переход на централизованную БД (для крупных компаний)

#### Необходимые изменения:
1. Установка PostgreSQL на сервер
2. Изменение `DATABASES` в `settings.py`:

```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': 'yandex_disk_manager',
        'USER': 'app_user',
        'PASSWORD': 'password',
        'HOST': 'server_ip',
        'PORT': '5432',
    }
}
Установка драйвера: pip install psycopg2-binary
